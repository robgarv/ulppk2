28 /*
29  * msgcell.h
30  *
31  * Created on: Dec 21, 2012
32  * Author: robgarv
33  *
34  * A "message cell" is a simple construct that facilitates inter process communications.
35  * A data object is logically associated with a POSIX semaphore. The data object is
36  * normally a memory mapped (an mmatom construct), so it can be safely accessed by multiple
37  * processes.
38  *
39  * A server process receives data through the data object. When it wants to
40  * receive data, the process calls msgcell_rcv, passing a check function.
41  * The check function checks the data object and if new data is present the call
42  * returns immediately. Otherwise, the calling process is blocked on a sem_wait.
43  *
44  * A client process writes data into the data object and calls msgcell_xmt to signal
45  * delivery of data. msgcell_xmt calls sem_post to signal a pending server process.
46  *
47  * This package makes no assumptions about the type or access methods of the data
48  * object. The calling application is responsible for creating/opening the data object,
49  * and closing it properly when access is no longer required.
50  *
51  * Data check functions, of course, incorporate knowledge of the data object and
52  * its access methods. These functions accept a single void* as input and return
53  * an int. A return of 0 (MSGCELL_NODATA_AVAIL) means no data available ... a return of
54  * 1 (MSGCELL_DATA_AVAIL) means data is available.
55  *
56  */
