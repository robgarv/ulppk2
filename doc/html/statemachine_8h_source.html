253 // Global transitions are useful for common error handling.
254 // For example, presume that all the states on the state/transition
255 // graph must handle the event EV_MYERROR, and that after handling
256 // the event the machine should land in the state MY_ERROR STATE.
257 // A common action list can be composed for this error event,
258 // Rather than register the event and action list for each
259 // defined state, call sm_register_global_transition to apply the
260 // transition to all the defined states. Restrictions:
261 // 1) ALL states must be required to handle the triggering event.
262 // 2) The same end state must be reached in all cases.
263 // 3) The same action list must be applied to the triggering event.
264 // 4) No other transition must be triggered by the event that triggers
265 // the global transition.
266 //
267 // Returns 0 if successful and non-zero otherwise.
268 //
269 int sm_register_global_transition(
270  SM_MACHINE *machinep, // state machine
271  const char *s2name, // name of the final state
272  const char* evname, // name of the triggering event
273  const char* actlist_name // name of the action list to execute
274 );
