\subsection{Detailed Description}
Memory Mapped Atom Utility. The mmatomx utility provides features for 
\begin{DoxyItemize}
\item Creating memory mapped atom files 
\item Dumping memory mapped file contents 
\end{DoxyItemize}

The dump option is pretty minimalistic and not very useful. Piping through od -\/c helps, but a proper octal dump utility needs to be incorporated.

mmmatomx -\/f /var/data/myfile.bin -\/d $\vert$ od -\/c

\subsubsection*{Command Line Use}

mmatomx -\/f \mbox{[}fullpath\mbox{]} -\/\mbox{[}dc\mbox{]}


\begin{DoxyItemize}
\item f -- full path to file (required)  
\item d -- perform dump 
\item c -- create file specified by -\/f option 
\item 
\end{DoxyItemize}

If the -\/c option is given, then the following additional options are available. 
\begin{DoxyItemize}
\item l -- Length of file/mem mapped region in bytes (default 4096) 
\item t -- Logical tag for memory mapped atom (default My\-Object) 
\item m -- Protection mode Protection mode R\-E\-A\-D\-\_\-\-O\-N\-L\-Y, W\-R\-I\-T\-E\-\_\-\-O\-N\-L\-Y, R\-E\-A\-D\-\_\-\-W\-R\-I\-T\-E (default R\-E\-A\-D\-\_\-\-W\-R\-I\-T\-E) 
\item M -- Map Flags\-: shared or private (default shared) 
\item p -- File permissions/access mode (octal) (default 664) 
\end{DoxyItemize}
