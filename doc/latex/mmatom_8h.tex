Lock the atom for read. 

This locks the entire range of bytes in the memory mapped region. The calling process will wait on the lock if necessary. Note that N processes can lock for reading, but if 1 process tries to lock for writing while the read lock is in effect, it will be blocked. Conversely, if 1 process has a write lock all other processes attempting to read or write lock will be blocked until the write lock is released.

Note that atoms implement mandatory locking.
